# 排班管理系统

本项目是一个本地优先的排班管理工具，支持 Web 开发模式和 Windows Electron 桌面版。它主要用于日常观察、快速试排和导出 Excel，最终排班结果仍可手动录入到钉钉 OA 等正式系统中。核心数据保存在本机 IndexedDB 中，不依赖后端服务，适合单人、单电脑、离线维护排班草稿。

## 功能概览

- 人员管理：新增、编辑、删除、导入、导出人员，支持重名区分
- 班次管理：维护班次、颜色、休息班次和显示顺序
- 排班管理：按月份排班，支持拖拽新增、移动、复制、排序和删除
- 人员视图：按人员查看月度排班和休息统计
- 节假日展示：标记法定节假日、调休工作日和周末
- 数据备份：导出/导入本地配置，可选择是否包含排班和已删除人员
- Excel 导出：导出月度排班表
- 桌面日志：记录主进程、渲染进程错误、崩溃和无响应诊断信息

## 下载与安装

GitHub Release 通常会提供 3 类 Windows 产物：

| 文件 | 适合场景 | 使用方式 |
| --- | --- | --- |
| `ban-<version>-setup.exe` | 日常安装使用 | 双击安装，按向导选择目录 |
| `ban-<version>-portable.exe` | 不想安装，直接运行 | 放到任意目录后双击运行 |
| `ban-<version>-green.zip` | 想自己管理完整目录 | 解压后运行目录内主程序 |

不确定选哪个时，优先使用 `setup.exe`。只是临时测试或放在移动盘中使用时，选择 `portable.exe`。

## 首次使用

1. 打开应用并登录。
2. 默认登录密码为 `0519`。
3. 忘记密码时可使用恢复码 `519519` 重置。
4. 进入“班次管理”确认或调整班次。
5. 进入“人员管理”录入人员和基础休息天数。
6. 进入“排班管理”选择月份并开始排班。
7. 初次配置完成后，到“配置管理”导出一份备份。

建议首次登录后尽快修改密码，并妥善保存恢复码。

## 排班说明

### 拖拽操作

- 从左侧人员列表拖到日期格子：新增排班
- 拖动已有人员标签到其他格子：移动排班
- 在同一个格子内拖动人员标签：调整顺序
- 拖动整格排班到其他格子：目标格会显示“移动 / 复制”动作区
- 拖到休息班次且超过剩余休息天数时，会弹出超额排休确认

业务规则：

- 同一人员同一天只能有一条排班
- 同一天内不能复制整格排班，只能移动到其他班次
- 已删除人员或已删除班次只保留历史记录，不允许继续排班

### 日期颜色

排班表会区分这些日期类型：

- 法定节假日
- 调休工作日
- 周六、周日

这些颜色只用于观察和提示，不会自动创建休息班次，也不会自动改变月休统计。

### 删除与历史数据

- 删除人员后，该人员不再参与后续排班
- 删除人员会清理今天及未来的排班，历史排班尽量保留
- 删除班次后，该班次不再用于后续排班，历史排班仍可回看
- 删除后重新新增同名人员或班次，会被视为新的数据

## 数据与备份

### 数据保存位置

系统默认使用 IndexedDB 保存业务数据：

- Web 模式：保存在当前浏览器本地数据中
- Electron 桌面版：保存在当前桌面应用环境中

因此，换电脑、重装系统、清理浏览器/应用数据，都可能导致本地数据丢失。重要数据请定期备份。

### 推荐备份时机

- 大批量导入前
- 修改大量人员、班次或排班前
- 升级版本前
- 更换电脑前
- 初始化系统前

### 配置导出

在“配置管理”中可以导出：

- 人员信息
- 班次信息
- 额外休息配置
- 排班记录
- 已删除人员相关历史数据

### 配置导入

导入配置时支持两种策略：

- 合并导入：保留当前数据，按 ID 合并导入内容
- 清空后恢复：先清空当前业务数据，再导入备份

如果需要完整还原旧备份，建议勾选“导入前清空现有数据”。如果备份里包含历史删除人员及其排班，还需要勾选“包含已删除人员”。

## 节假日数据

系统内置中国节假日数据，离线也能展示。配置管理中支持：

- 同步指定年份节假日
- 导入节假日 JSON 文件
- 恢复内置节假日数据
- 清理远程同步或手动导入来源

节假日来源优先级为：手动导入 > 远程同步 > 内置数据。

## 登录与安全

- 本系统使用本地口令登录，不是服务端账号体系
- 默认登录密码：`0519`
- 默认恢复码：`519519`
- 修改密码、重置密码、初始化系统后，已有登录会话会失效
- 自定义密码以哈希形式保存在本地
- 支持无操作自动退出和会话续期

该机制适合本地单机访问控制，不适合作为多人在线权限系统。

## 日志与问题定位

桌面版会记录关键错误和崩溃诊断，便于定位问题：

- 主进程未捕获异常
- 主进程未处理 Promise
- 渲染进程页面错误和未处理 Promise
- `console.error`
- preload 异常
- 页面加载失败
- 渲染进程崩溃或被系统终止
- GPU/子进程异常退出
- 窗口无响应和恢复响应
- 本地 crash dump 状态

日志默认写入桌面应用的 `userData/logs` 目录。Windows 下通常类似 `%APPDATA%\班\logs`，目录名会跟随应用名称“班”。如果界面没有提供日志导出入口，可以手动进入该目录查找 `app-*.log` 和 `error-*.log`。

崩溃 dump 由 Electron `crashReporter` 本地采集，当前配置为不上传服务器。日志导出能力会追加 `crash-diagnostics.json`，其中包含 crash dump 目录和最近 dump 文件列表。

反馈问题时建议提供：

- 软件版本
- Windows 版本
- 使用的是安装版、便携版还是绿色版
- 复现步骤
- 截图或录屏
- `logs` 目录中的 `app-*.log` 和 `error-*.log`
- 如果存在 crash dump，同时保留对应 dump 文件

## 常见问题

### 换电脑后数据为什么没有自动出现？

数据默认保存在本机，不会自动同步。请先在旧电脑导出配置，再在新电脑导入。

### 节假日会自动改变排班吗？

不会。节假日只影响页面展示，不会自动排休，也不会修改统计规则。

### 为什么同一天不能给同一个人排多个班？

系统按“同一人员同一天一条排班”设计，用于避免统计和导出出现歧义。

### 便携版和绿色版有什么区别？

便携版是单个 exe，适合直接运行；绿色版是 zip，适合解压后保留完整目录结构。

## 开发者说明

普通使用者可以忽略本节。

### 技术栈

- Vue 3
- TypeScript
- Element Plus
- IndexedDB
- Electron 39
- Vite
- electron-builder

### 环境要求

- Node.js 18 及以上
- npm 9 及以上
- Windows x64

### 安装依赖

```bash
npm ci
```

如果只是本地开发，也可以使用：

```bash
npm install
```

### 开发命令

| 命令 | 说明 |
| --- | --- |
| `npm run dev` | 启动 Web 开发模式 |
| `npm run dev:electron` | 启动 Electron 开发模式 |
| `npm run type-check` | 检查 Vue/前端类型 |
| `npm run type-check:electron` | 检查 Electron 主进程和 preload 类型 |
| `npm run lint` | 运行 ESLint |

### 构建与打包

| 命令 | 产物/用途 |
| --- | --- |
| `npm run build:desktop` | 只构建桌面应用所需的 `dist` 和 `dist-electron`，不生成安装包 |
| `npm run pack:dir` | 生成未压缩的本地目录，适合快速验证打包后行为 |
| `npm run build:portable` | 生成便携版 `ban-<version>-portable.exe` |
| `npm run build:setup` | 生成安装版 `ban-<version>-setup.exe` |
| `npm run build:green` | 生成绿色版 `ban-<version>-green.zip` |
| `npm run build:all` | 同时生成 portable、setup、zip |

打包产物输出到：

```text
release/<version>/
```

`build:portable`、`build:setup`、`build:green`、`build:all` 和 `pack:dir` 会在打包前执行：

1. 清理当前版本的 `release/<version>/`
2. 更新内置节假日数据
3. 执行 `build:desktop`
4. 运行 `electron-builder`

### 发布文件命名

正式发布文件命名规则：

- `ban-<version>-setup.exe`
- `ban-<version>-portable.exe`
- `ban-<version>-green.zip`
